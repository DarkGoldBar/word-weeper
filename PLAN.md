# PLAN.md

## 1. 项目目标

实现一个本地单机双人网页游戏“扫词”。

两个玩家轮流输入词语。系统计算输入词与15个胶囊词的语义相似度，最高值对应的胶囊视为本回合命中。

所有计算、状态和数据均保存在浏览器本地，不使用网络连接，不考虑反作弊。

## 2. 技术栈

* Vue 3
* TypeScript
* Vite
* Pinia
* windsurf
* Transformers.js
* Web Worker
* IndexedDB 或 localStorage
* CSS Media Query 或响应式宽度判断

## 3. 游戏布局

页面分为三部分：

1. 顶部：游戏状态
2. 中部：15个胶囊和输入框
3. 底部：猜测历史表格

胶囊顺序固定：

* 前5个：红色目标词
* 中间5个：灰色中立词
* 后5个：蓝色目标词

## 4. 胶囊状态

### 红色胶囊

* 初始隐藏文字
* 首次命中后公开文字
* 已公开后仍参与相似度计算
* 再次命中不产生进度

### 灰色胶囊

* 初始直接显示文字
* 始终参与相似度计算
* 命中后不产生进度
* 可重复命中

### 蓝色胶囊

* 初始隐藏文字
* 首次命中后公开文字
* 已公开后仍参与相似度计算
* 再次命中不产生进度

## 5. 回合规则

1. 当前玩家输入一个词。
2. 生成输入词的 embedding。
3. 计算输入词与15个胶囊词的 cosine similarity。
4. 取最高值对应的胶囊。
5. 该胶囊视为本回合命中。
6. 若首次命中隐藏的红色或蓝色胶囊，则公开文字。
7. 无论是否产生进度，回合结束后切换玩家。
8. 不排除已公开胶囊。
9. 不重新寻找未公开胶囊。
10. 最高值始终等于实际命中结果。

## 6. 胜利条件

* 红色5个目标词全部公开，红方获胜。
* 蓝色5个目标词全部公开，蓝方获胜。
* 胜利后禁止继续输入。
* 提供重新开始按钮。

## 7. 玩家切换

使用本地轮流模式：

```text
红方 → 蓝方 → 红方 → 蓝方
```

页面必须明确显示：

* 当前玩家
* 红方已公开数量
* 蓝方已公开数量
* 当前游戏状态

## 8. 输入规则

* 输入不能为空。
* 自动去除首尾空格。
* 禁止在模型未加载完成时提交。
* 禁止在相似度计算期间重复提交。
* 可使用 Enter 提交。
* 提交后清空输入框。
* 保留重复输入，重复输入仍正常计算。

## 9. 相似度表格

每次猜测新增一行。

表格列：

* 玩家
* 输入词
* 红1至红5
* 灰1至灰5
* 蓝1至蓝5

显示要求：

* 相似度显示为百分比。
* 每行最高值加粗。
* 最高值对应单元格增加命中样式。
* 表格按猜测时间排列。
* 最新记录放在最上方或最下方，保持一致。
* 表头顺序必须与胶囊顺序一致。

## 10. Embedding 模型

优先使用支持中文的浏览器端 embedding 模型。

要求：

* 使用 Transformers.js。
* 模型运行放入 Web Worker。
* 优先使用 WebGPU。
* 不支持 WebGPU 时回退到 WASM。
* 模型只初始化一次。
* 棋盘生成后预先计算15个胶囊词向量。
* 玩家每次输入只计算一次向量。
* 所有向量统一归一化。
* cosine similarity 使用归一化向量点积计算。

## 11. 本地数据

初始版本使用内置词库。

词库至少包含：

```ts
interface WordEntry {
  id: string
  word: string
  category: string
}
```

棋盘数据：

```ts
type CapsuleTeam = "red" | "neutral" | "blue"

interface Capsule {
  id: string
  word: string
  team: CapsuleTeam
  revealed: boolean
  embedding?: Float32Array
}
```

猜测记录：

```ts
interface GuessRecord {
  id: string
  player: "red" | "blue"
  input: string
  similarities: number[]
  hitIndex: number
  createdAt: number
}
```

## 12. 棋盘生成

每局随机选择15个不同词。

分配方式：

* 红色5个
* 灰色5个
* 蓝色5个

要求：

* 不允许重复词。
* 灰色词初始公开。
* 红蓝词初始隐藏。
* 暂不实现复杂语义平衡算法。
* 暂不使用大语言模型生成词语。

## 13. 本地存储

保存以下状态：

* 当前棋盘
* 当前玩家
* 胶囊公开状态
* 猜测历史
* 游戏结果

刷新页面后恢复当前游戏。

重新开始时清除旧状态并生成新棋盘。

## 14. 响应式界面

通过页面宽度区分PC端和移动端。

### PC端

* 15个胶囊尽量单行显示。
* 历史表格允许横向滚动。
* 输入框和按钮横向排列。
* 页面内容保持居中。

### 移动端

* 胶囊按红、灰、蓝分为三行。
* 每组5个胶囊自适应宽度。
* 输入框和按钮可上下排列。
* 历史表格横向滚动。
* 固定第一列或前两列，便于查看输入词。
* 点击区域满足触屏操作。

建议断点：

```text
小于768px：移动端
大于等于768px：PC端
```

## 16. 核心模块职责

### gameEngine

负责：

* 相似度比较
* 命中判定
* 胶囊公开
* 玩家切换
* 胜负判定

### embedding

负责：

* 初始化模型
* 生成 embedding
* 返回归一化向量

### boardGenerator

负责：

* 随机选词
* 分配红灰蓝胶囊
* 初始化公开状态

### storage

负责：

* 保存游戏
* 恢复游戏
* 清除游戏

## 17. 开发顺序

1. 初始化 Vue 3 和 TypeScript 项目。
2. 定义数据结构。
3. 完成静态响应式页面。
4. 完成本地双人回合逻辑。
5. 使用临时随机相似度验证流程。
6. 接入 Transformers.js。
7. 将模型推理移入 Web Worker。
8. 完成真实 cosine similarity。
9. 完成历史表格。
10. 完成本地存储和恢复。
11. 完成移动端适配。
12. 增加错误处理和加载状态。
13. 进行完整游戏测试。

## 18. 验收标准

* 页面可随机生成15个胶囊。
* 红蓝词初始隐藏，灰色词初始公开。
* 两名玩家可以本地轮流操作。
* 每次输入可生成15个相似度。
* 最高值对应胶囊始终为实际命中目标。
* 已公开词仍可重复命中。
* 每次提交后自动切换玩家。
* 红方或蓝方全部公开后正确结束游戏。
* 历史表格正确显示所有结果。
* 刷新页面后可以恢复游戏。
* PC端和移动端均可正常操作。
* 整个游戏不依赖服务端和网络API。
